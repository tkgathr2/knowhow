============================================================
第1章 背景（詳細）
============================================================

現在、当社ではDevinおよびClaude CodeなどのAIエージェントを活用し、
設計・実装・デバッグ・検証・デプロイまでを実行している。

しかし、AIエージェントはセッション単位で動作する構造であり、
セッション終了と同時に内部コンテキストが消失する。

この構造的制約により、同一エラーの再発、知見の非資産化、
ルール逸脱リスク、再現性の欠如、AIが成長しないという問題が継続的に発生している。

本プロジェクトは、開発ログをAIで構造化・Embedding化し、
類似検索可能なナレッジDBを構築することで、
AIを「セッション型」から「外部記憶を持つ経験蓄積型」へ進化させる。

これは単なるログ保存ではない。 高木OSの中核である。

============================================================
第2章 要件定義
============================================================

■ 目的
・同一失敗の再発防止
・デバッグ時間50%以上削減
・憲法常時優先適用
・AI運用知見の資産化

■ KPI
・再発エラー率50%削減
・検索ヒット有効率70%以上
・憲法参照率100%
・Search API P95 3秒以内

■ 機能要件
・非同期Ingest処理
・JSON構造化要約
・ハイブリッド検索
・フィードバックループ（Beta更新）
・HNSWインデックス
・セキュリティ三層防御
・障害時再試行設計
